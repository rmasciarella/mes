## Toolchain: Vite+ (`vp`)

This project uses **Vite+**, a unified toolchain wrapping Vite, Rolldown, Vitest, tsdown, Oxlint, and Oxfmt. The global CLI is `vp`. **Do not use pnpm/npm/yarn directly.**

- `vp run <script>` runs package.json scripts (NOT `vp <script>`, which invokes Vite+ built-ins)
- `vp dlx <pkg>` / `vpx <pkg>` replaces npx
- Import from `vite-plus` and `vite-plus/test`, not from `vite` or `vitest`
- Do not install Vitest, Oxlint, Oxfmt, or tsdown separately

## Key Commands

```bash
vp run dev              # Start all dev servers (web :3000, server :5000)
vp run fix              # Format + lint + type-check (runs on git commit)
vp run test             # Run all tests
vp run build            # Build all packages

vp run db:dev:start     # Start local PostgreSQL via Docker
vp run db:dev:stop      # Stop local PostgreSQL
vp run db:push          # Push schema changes to DB
vp run db:generate      # Generate migration files
vp run db:migrate       # Run pending migrations
vp run db:studio        # Open Drizzle Studio GUI

vp run docker:up        # Start full stack in Docker
vp run docker:up:build  # Rebuild + start full stack
vp run auth:secret      # Generate Better Auth secret
vp run ui               # Add shadcn/ui components
```

## Architecture

### Request Flow

1. **Hono server** (`apps/server/src/index.ts`) applies CORS, logger middleware, mounts Better Auth at `/auth/*`
2. **oRPC handlers** serve RPC at `/rpc/*` and OpenAPI/Scalar docs at `/docs/*`
3. **Context factory** resolves session via `auth.api.getSession()`, creates mediator and event dispatcher
4. **Procedures**: `publicProcedure` (no auth) and `protectedProcedure` (requires session, throws UNAUTHORIZED)
5. **Composition root** (`apps/server/src/infrastructure/`) wires repositories, event dispatcher, mediator

### Domain-Driven Design (`packages/core`)

- `iam/domain/` - User entity, UserId value object, domain events
- `iam/application/` - Query handlers, repository ports, auth ports
- `shared-kernel/` - Base DomainEvent class
- `port/` - EventDispatcher and Mediator port interfaces
- `architecture.test.ts` - Fitness functions validating layer boundaries

### Database (`packages/db`)

- **Drizzle ORM** with PostgreSQL via `postgres-js` driver
- Schema uses `snake_case` casing convention (auto-converts from camelCase)
- Relations defined via `defineRelations()` (Drizzle v2 style)
- IAM tables generated by Better Auth CLI into `src/schema/iam/tables.ts`
- Repositories in `src/repositories/` provide data access patterns
- Migrations auto-run on production startup

### Authentication (Better Auth)

- Self-hosted, cookie-based sessions with 5-minute cache
- Email/password enabled by default
- Cookie: `SameSite=Strict` (prod), `SameSite=None` (dev for Safari cross-port)
- Server and web **must share the same host** with path-based routing (SameSite constraint)

### Frontend (`apps/web`)

- **TanStack Start** with file-based routing, TanStack Router + Query
- Routes use locale prefix: `/{-$locale}/` (optional param via Paraglide)
- API calls via oRPC client with TanStack Query adapter
- UI: shadcn/ui + Radix primitives, Tailwind CSS v4, next-themes

### i18n (Paraglide.js)

- Compiled at build time (not runtime), outputs to `packages/i18n/src/paraglide/`
- All i18n route variants are pre-rendered at build

## Environment

All env vars live in `packages/env/.env` (copy from `.env.example`). Loaded via `dotenvx`.

Required: `VITE_SERVER_URL`, `VITE_WEB_URL`, `DATABASE_URL`, `BETTER_AUTH_SECRET` (32+ chars).

## Conventions

- Package scope: `@tsu-stack/*`
- Internal imports use `#@/*` path aliases
- oRPC routers nested under `packages/api/src/routers/` (e.g., `health/`, `iam/`)
- Pre-commit hook runs `vp check` automatically via Vite+ prepare hook
- CI: use `voidzero-dev/setup-vp@v1` GitHub Action
